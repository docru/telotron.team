# Аджайл — процесс Telotron (регламент)

**Статус:** согласовано с директором (май 2026)  
**Назначение:** **достаточный** процесс без перегруза — ритм разработки, единый backlog, связь с вехами и ростом. Не «Scrum по учебнику».

**Связь:** [календарь спринтов 2026](Аджайл%20—%20календарь%20спринтов%202026%20(до%20конца%20лета).md) · [План до 31.12.2026](План%20до%2031.12.2026%20—%20вехи%20и%20цели%20по%20периодам.md) · [план-доработки-период-0](../../../_telotron.ru/docs/Техдок/00-мета/план-доработки-период-0.md)

---

## 1. Зачем (цели процесса)

1. Согласовать работу **всех разработчиков** (сейчас один, позже несколько).
2. Простой **документооборот**: одна задача = один тикет, всё внутри тикета.
3. **Ритм 2 недели** без вечных рывков (capacity ~**10–15 ч/нед** на продукт).
4. Достижение **целей вех** (01.06, 01.08, 100 платников, доход к концу лета).

**Провал процесса (2–3 мес.):** к **концу сентября** нет устойчивого **~200 тыс. ₽/мес личного дохода** основателя (доход проекта − все расходы; до найма = ЗП на уход с найма) — процесс не должен съедать время на delivery и outreach.

---

## 2. Роли

| Роль | Кто | Зона |
|------|-----|------|
| **Директор** | 01 | Направления **на квартал**, вехи, go/no-go (деплой, РКН, outreach), veto scope |
| **Product Owner** | 02 Маркетолог | **Backlog**, тема спринта, приоритет; **insight brief**; регулярное выяснение болей — **ключ к росту** |
| **Delivery** | 03 Разработчик | Оценка capacity, delivery, техдолг |
| **Ведущий процесса** | Пока **директор** (15 мин/нед) | Календарь спринта, напоминание planning/retro, «задачи из backlog» |
| **QA / архитектор** | 09 / 10 | **По приглашению** (billing, MAX, ADR) |

Пока команда из одного человека — **те же правила**, явно переключать шляпы: PO (маркетолог) vs директор vs разработчик.

**Go/no-go** (чеклист A1–A13, РКН, старт outreach) — **не церемония спринта**, решение **директора** по документу ([план периода 0](../../../_telotron.ru/docs/Техдок/00-мета/план-доработки-период-0.md), [уведомление РКН](../../../_telotron.ru/docs/Бизнес-требования/04-комплаенс-и-аудит/уведомление-роскомнадзор-оператор-пд.md)).

## 3. Артефакты (только нужное)

| Артефакт | Что это |
|----------|---------|
| **Product backlog** | [`Команда/backlog/`](../../backlog/README.md) — **`идеи/`** → **`бэклог/`** → **`в-работе/`** → **`сделано/`**; **`отменено/`** — только директор |
| **Идеи** | [`идеи/`](../../backlog/идеи/) — наброски **без** спринта и без T-XXX |
| **Тикет** | Один файл в `бэклог/`…; **папка = статус**; перенос при взятии/закрытии |
| **Тема спринта** | Одна **крупная тема** + **1–3 подзадачи** (не длинный список мелочей) |
| **Insight brief** | **Раз в 2 недели** перед planning: боль → кого затронуло → что в backlog; если инсайтов нет — явно «нет новых» |
| **Итог спринта** | Письменно в backlog/тикете: что закрыто, ссылки (**demo как встреча не проводим**) |

**Не используем:** правило «только с evidence»; приоритет — PO + вехи директора.

**Задачи:** delivery — **только** из **`бэклог/`** (или P0 hotfix). Наброски — в **`идеи/`**; в спринт после созревания. Сделали мимо backlog — разбор на **retro**.

## 4. Церемонии (потолок **≤ 2 ч / спринт**)

| Событие | Когда | Длительность | Содержание |
|---------|-------|--------------|------------|
| **Insight brief** | Чт–пт перед planning | 20–30 мин | PO: боли, интервью, M14 → черновик в backlog |
| **Planning** | Пн, день 1 спринта | 30–45 мин | Тема спринта + 1–3 тикета; capacity; директор — только если меняется веха |
| **Async-статус** | Каждые 1–2 раб. дня | 3 мин | В тикете или один тред: вчера / сегодня / блокер |
| **Retro** | Последний раб. день спринта | 20–30 мин | Что мешало; мимо backlog; почему не влезло → **переприоритизация** |

**Не делаем:** weekly sync (пока один человек); отдельная demo; daily standup 15 мин; refinement 1 ч/нед; story points / burndown.

**Спринт 0 (25–31.05):** только **planning** + **retro**; отработать ритуал на теме «закрытие периода 0».

---

## 5. Definition of Done (задача)

1. Файл тикета в папке **`сделано/`**; в шапке статус **`done`** + итог в журнале.
2. Код **merged** (если была разработка).
3. Проверка на **stage** по умолчанию; **prod** — по вехам / go/no-go директора, не каждая мелкая задача.

**В работу:** исполнитель переносит тикет из **`бэклог/`** в **`в-работе/`** (`in_progress`). **Отмена:** только директор → **`отменено/`**.

**Незавершённое:** на границе спринта — **почему не влезло** + **переприоритизация** (файл остаётся в `в-работе/` или возвращается в `бэклог/` — запись в журнале).

---

## 6. Отпуск 06.07 – 12.07

- **Новый scope спринта не стартуем**; planning/review в неделю отпуска **не проводим**.
- **On-call нет** — директор **недоступен**.
- До отпуска (спринт **3**, короткий): только то, что реально закрыть; prod в **стабильном** состоянии; риски P0 — закрыть **до 05.07**.

---

## 7. Чего нет в процессе (намеренно)

- Story points, velocity, burndown  
- Жёсткие user story / INVEST  
- Обязательная demo  
- Weekly sync (до появления 2+ человек в delivery)  
- Evidence-only backlog  
- Привязка go/no-go к retro  

---

## 8. Успех процесса (к 31.07)

| Признак | Цель |
|---------|------|
| Ритм | ≥ **3** спринта с planning + retro + письменным итогом |
| Backlog | Нет «скрытых» задач вне backlog |
| Инсайты | ≥ **1** тема спринта явно из insight brief |
| Capacity | ≥ **70%** product-часов на тему спринта, мимо backlog — разбираем на retro |
| Бизнес | **~200k личного дохода** (ориентир конец сентября), вехи **01.08** / **01.10** (100 платников) — вне спринта; PO: [T-004](../../backlog/бэклог/T-004-оценка-привлечения-воронка.md), [T-005](../../backlog/бэклог/T-005-матрица-функция-тариф.md) |
