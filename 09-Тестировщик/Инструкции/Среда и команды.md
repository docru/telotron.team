# Среда и команды

## Домены (локально)

| Зона | Домен (пример) | Назначение |
|------|----------------|------------|
| Pro | `pro.telotron.test` | PWA тренера, `/workspace` |
| Client | `client.telotron.test` | PWA клиента |

| Публичный | `telotron.test` | Витрина, хаб `/i/{token}` |

Точные имена — у **сисадмина** или в `_telotron.ru/docs/Техдок/02-зоны/домены-маршруты-сессии.md`.

## Docker (обязательно для тестов и сборки)

Рабочий каталог приложения: **`_telotron.ru/`** (где `compose.yaml`).

**Не запускать** на хосте `php`, `artisan`, `composer`, `npm` для этого проекта — только в контейнере **`laravel.test`**, пользователь **`sail`**:

```bash
cd _telotron.ru
docker compose up -d
docker compose exec -u sail laravel.test php artisan test
docker compose exec -u sail laravel.test php artisan test --filter=SchedulingApiTest
docker compose exec -u sail laravel.test npm run test:ts
docker compose exec -u sail laravel.test npm run build
docker compose exec -u sail laravel.test npm run test:e2e
```

**E2E (Playwright):** в `.env` нужны `E2E_ENABLED=true` и `E2E_SECRET` (только local). Перед первым прогоном: `npm install`, `npx playwright install chromium`, `npm run build`. Подробности: `_telotron.ru/e2e/README.md`.

При вызове из CI/агента без TTY можно добавить `-T` после `exec`.

## Перед ручным тестом PWA

1. Убедиться, что контейнеры подняты.
2. После изменений фронта — **`npm run build`** от **`sail`** (иначе права на `public/build/` и устаревшие чанки).
3. В браузере при «залипании» старого UI: жёсткое обновление или сброс данных сайта / SW для зоны Pro или Client.
4. Сравнить `build` в `config/version.php` с ответом `GET /version.json` на домене зоны.

## Учётные записи

- Тестовые пользователи и сиды — согласовать с **разработчиком** / **сисадмином** (не хранить пароли в открытых чеклистах; для Passkey — отдельные тестовые устройства или стенд-only flow).
- Admin: первый супер-админ из env (см. техдок RBAC / Filament).

## Что проверять при сбое «не открывается»

1. `docker compose ps` — сервисы healthy.
2. DNS / hosts для `*.telotron.test`.
3. Не смешивать сессии Pro и Client в одной вкладке без понимания (куки по поддоменам не шарятся — это норма).
