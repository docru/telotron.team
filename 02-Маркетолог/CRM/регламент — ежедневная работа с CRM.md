# Регламент · ежедневная работа с CRM

**Для кого:** директор (касания, follow-up).  
**Сколько времени:** **15–20 мин** рутина + блок outreach по плану дня (**5–7 касаний/нед** в активные недели).  
**Файлы:** [CRM/](.) · `contacts.csv` · `touches.csv`

---

## Принципы (чтобы не путаться)

1. **Сначала статус, потом действие** — перед сообщением открыть строку контакта и проверить `status` + `channel`.
2. **Один канал — один токен** — перед отправкой reg свериться с [таблицей каналов](README.md#4-каналы-и-токены).
3. **Пилот = анкета → reg** — ссылку `/i/…` **не** отправлять до `survey_done` (кроме редкого исключения по решению директора с пометкой в `notes`).
4. **Касание = строка в `touches.csv`** — историю не править, только добавлять.
5. **Reg в prod** — статус `connected` ставит директор после проверки или PO на пятничной сверке.

---

## Утро · 10 мин · «что сегодня»

### Шаг 1. Открыть очередь

В `contacts.csv` отфильтровать:

1. **`next_action_date` ≤ сегодня** — главная очередь дня.
2. **`status` = `link_sent`** и link_sent > **2 дней** назад (по последнему touch) — мягкий follow-up.
3. **`status` = `survey_sent`** и > **3 дней** — напоминание об анкете.

Записать себе **3–5 имён** на сегодня (не больше реалистичного).

### Шаг 2. Дозаписать вчера

Если вчера были звонки/лички без строки в `touches.csv` — **сначала** дописать, потом новые касания.

## День · outreach · «делаю касание»

Перед **каждым** контактом — мини-чеклист:

```text
□ Открыл строку contacts.csv по id
□ channel и token совпадают с ситуацией (не VK-токен в WhatsApp)
□ status подсказывает, ЧТО делать (см. таблицу ниже)
□ После действия — touch + обновил status / next_action_*
```

### Что делать по статусу

| `status` | Действие сегодня | После успеха |
|----------|------------------|--------------|
| `new` | Первое сообщение / просьба о 15 мин | → `contacted` |
| `contacted` | 2-й пинг или назначить созвон | → `call_scheduled` |
| `call_scheduled` | Созвон: пилот, без оплат до 01.08 | → `pilot_agreed` или `pause`/`lost` |
| `pilot_agreed` | Отправить **входную анкету** | → `survey_sent` |
| `survey_sent` | Напоминание об анкете | → `survey_done` когда заполнена |
| `survey_done` | **Reg** (токен канала) + [инструкция PDF](../Инструкции/Онбординг%20—%20инструкция%20для%20тренеров.pdf) | → `link_sent` |
| `link_sent` | Follow-up: reg прошёл? | → `connected` когда reg есть |
| `connected` | Помочь дойти до первого **реального** клиента | → `activated` |
| `activated` | Поддержка, раз в 1–2 нед — «как идёт» | ждать 30 д от `reg_at` |
| `retained_30d` | Ближе к концу пилота — **выходная анкета** | → `exit_survey_sent` |
| `exit_survey_sent` | Напоминание | → `exit_survey_done` |
| `pause` | Не трогать до `pause_until` | — |

### Запись касания

В `touches.csv` одна строка:

```text
2026-06-21,krasnodar-maxftrener,personal_circle,jPp1HN,link_sent,dialog,director,отправил анкету и PDF после survey_done
```

Обновить в `contacts.csv`: `status`, `next_action_date` (+1–3 дня), `next_action` одной фразой.

## Типовые ошибки

| Ошибка | Как избежать |
|--------|----------------|
| Отправил reg до анкеты | Чеклист: `pilot_agreed` → только `survey_sent`, не `link_sent` |
| Один токен на всех | Поле `channel` при создании контакта |
| Статус `activated` без реального клиента | Только после принятого инвайта, не тестового |
| Нет `next_action_date` | После каждого touch обязательно |
| Правка старых строк touches | Запрещено; исправление — новая строка `note` |
