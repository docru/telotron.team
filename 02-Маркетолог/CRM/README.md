## 1. Воронка (основной статус)

Линейный путь. **Один контакт = одна строка** в `contacts.csv`, текущий этап — поле `status`.

| # | `status` | Смысл | Типичное следующее действие |
|---|----------|--------|-----------------------------|
| 1 | `new` | Контакт есть, касания не было | Первое сообщение / запрос на созвон |
| 2 | `contacted` | Было касание, созвон не назначен | Назначить созвон или 2-й пинг |
| 3 | `call_scheduled` | Созвон в календаре | Провести созвон |
| 4 | `pilot_agreed` | **Договорились об участии в пилоте** | Отправить **входную анкету** |
| 5 | `survey_sent` | Ссылка на анкету отправлена | Дождаться / напомнить |
| 6 | `survey_done` | Анкета заполнена | Отправить **reg + инструкцию** (только свой токен канала) |
| 7 | `link_sent` | Ссылка reg отправлена, reg ещё нет | Follow-up 24–48 ч |
| 8 | `connected` | **Reg Pro** в prod (подключён к пилоту) | Сопровождение до первого клиента |
| 9 | `activated` | **Первый реальный клиент** (инвайт принят) | Сопровождение использования |
| 10 | `retained_30d` | **30 дней от reg** + критерий активности (§3) | К концу пилота — выходная анкета |
| 11 | `exit_survey_sent` | Выходная анкета отправлена | Напомнить |
| 12 | `exit_survey_done` | Выходная анкета заполнена | Архив / партнёрское ядро / оплата с 01.08 |
| 13 | `paying` | Первая оплата (с **01.08**, когда commerce жив) | — |

**Побочные статусы** (не «прогресс вперёд»):

| `status` | Когда |
|----------|--------|
| `pause` | Нет ответа после 2 попыток; вернуться через `pause_until` |
| `lost` | Явный отказ или нецелевой контакт |

> **Порядок пилота:** сначала **анкета**, потом **reg** — чтобы первичный опрос не искажался знакомством с продуктом.

> **Activation в CRM** = **первый реальный клиент** (упрощение для операционки). В [плане продвижения §1.3](../План%20продвижения%20—%20этап%201%20Пилот.md) activation шире (клиент + календарь/план за 14 дней) — для **цели 1** и отчётов PO можно сверять отдельно, не смешивая поле `status`.

## 2. Параллельные вехи (не заменяют `status`)

Отдельные колонки в `contacts.csv` — могут заполниться **раньше** поздних этапов воронки.

| Поле | Смысл |
|------|--------|
| `referral_at` | Дата первого приведённого реферала (reg по его ссылке) |
| `referral_count` | Сколько рефералов всего |
| `partner_core` | `no` / `candidate` / `yes` — ядро партнёров 5–10 (июль) |
| `first_payment_at` | Первая оплата (с 01.08; частично партнёрскими — когда модуль жив) |

---

## 3. Критерий `retained_30d`

- **Отсчёт:** **30 календарных дней от `reg_at`** (дата reg в prod), не от activation.
- **Условие (достаточно одного):**
  - есть **запланированные** тренировки впереди, **или**
  - **активность в журнале** за последние **2 дня** (Client/Pro — по смыслу «ведёт работу»).
- **`retained_verified`:** `auto` (из prod, когда будет отчёт) / `manual` (созвон, скрин) / `pending`.

## 5. Файлы

| Файл | Назначение |
|------|------------|
| `contacts.csv` | Реестр тренеров (1 строка = 1 человек) |
| `touches.csv` | Журнал касаний (**только добавление** строк) |
| `sources.csv` | Коннекторы: пасторы, знакомые (**не целевики**) — см. §6 |
| `interviews-index.md` | Связь `contacts` ↔ обобщения интервью |
| `channels.yaml` | Справочник канал → токен (дубль §4 для скриптов) |
| `partners.yaml` | Outreach-партнёры (не тренеры) · личная reg на человека |
| `weekly/` | Недельные сводки ([шаблон](weekly/_шаблон-сводка.md)) |

### `contacts.csv` — ключевые колонки

```text
id,name,city,channel,token,source_id,segment,warmth,status,
survey_sent_at,survey_done_at,reg_at,activated_at,retained_30d_at,
exit_survey_sent_at,exit_survey_done_at,referral_at,referral_count,
partner_core,first_payment_at,retained_verified,
next_action_date,next_action,handle,pause_until,notes
```

### `touches.csv`

```text
date,contact_id,channel,token,action,outcome,by,note
```

`action`: `message` · `call` · `call_scheduled` · `survey_sent` · `survey_done` · `link_sent` · `instruction_sent` · `follow_up` · `exit_survey_sent` · `note`

## 6. Коннекторы (`sources.csv`)

**Пасторы и знакомые — не целевики.** Их нет в `contacts.csv`; ведём обзвон в `sources.csv`:

```text
id,name,city,type,priority,handle,status,next_action_date,next_action,notes
```

| `type` | Пример |
|--------|--------|
| `church` | пастор Филипп — даст контакты тренеров |
| `personal` | Герман, Аня — личный круг |
| `interview_pool` | мета-источник «база интервью» |
| `partner_outreach` | продажник / outreach-партнёр (своя reg, см. `partners.yaml`) |

После обзвона: новые **тренеры** → строка в **`contacts.csv`** с `source_id` пастора/коннектора.

## 6. Сверка с prod

| Частота | Действие |
|---------|----------|
| **Пятница** (30 мин с PO) | Reg по токенам в Filament → `reg_at`, `connected` |
| По факту | «Клиент принял инвайт» → `activated` |
| **reg_at + 30 дней** | Проверка retention → `retained_30d` |
| С **01.08** | `first_payment_at`, `paying` |
