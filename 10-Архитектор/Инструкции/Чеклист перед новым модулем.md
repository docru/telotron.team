# Чеклист перед новым модулем

Проходить **до** merge основной реализации (архитектор или разработчик в режиме архитектора).

## 1. Продукт и граница

- [ ] Модуль имеет **одного владельца** в `app/Modules/<Name>/`.
- [ ] В scope MVP явно: что **входит / не входит** (ссылка на Требования-MVP-v1 или бизнес-модуль).
- [ ] Нет дублирования существующего модуля (не второй «календарь» в Invites).

## 2. API

- [ ] Endpoints описаны в **api-http-контракт-mvp** (§4.1x или новый подраздел).
- [ ] Мутации с JSON-телом: **idempotency_key** где нужен outbox/retry.
- [ ] Ошибки: 422 / 403 / 404 / 409 с **problem+json**, не ad-hoc JSON.
- [ ] Pro / Client / Admin: кто какой endpoint видит; нет «утечки» admin-only в Client.

## 3. Данные

- [ ] Миграции: таблицы в владении модуля; FK — осознанно, не циклы.
- [ ] Индексы под списки и календарь (trainer_id + time range).
- [ ] Soft delete / audit — если затрагивает ПД или расследования поддержки.

## 4. Backend

- [ ] Контроллеры тонкие; логика в `Modules/<Name>/`.
- [ ] Нет прямых вызовов Eloquent **чужого** модуля из контроллера — через сервис/контракт модуля.
- [ ] Jobs и очереди: retry-safe, идемпотентны где возможно.

## 5. Frontend (если есть UI)

- [ ] Страницы в `app/pro` или `app/client`, общее — в `shared/`.
- [ ] API через `TelotronApi` / единый слой, не raw fetch с разными правилами.
- [ ] Client: offline-политика модуля согласована (read-only vs outbox vs online-only).
- [ ] Lazy routes для тяжёлых экранов.

## 6. Зоны и PWA

- [ ] Изменения shell / SW / manifest → bump **`build.pro`** или **`build.client`** по зоне.
- [ ] `theme-*` и домен не смешаны.

## 7. Тесты

- [ ] Feature API на happy + 1–2 негативных кейса.
- [ ] Vitest на парсеры/утилиты модуля при наличии.
- [ ] Критичный сценарий добавлен в чеклист тестировщика при необходимости.

## 8. Документы

- [ ] Канон или ADR обновлены **в том же цикле**, что merge (не «потом»).
- [ ] План-реализации-mvp: галочка этапа — если применимо.

## Вердикт

| Результат | Действие |
|-----------|----------|
| Все пункты OK | Merge |
| Пробелы некритичные | Merge + issue в backlog архитектора |
| Пробелы в API/границах/зонах | **No merge** до ADR или правки |
