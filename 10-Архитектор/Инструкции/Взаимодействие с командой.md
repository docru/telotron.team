# Взаимодействие с командой

## Роли в архитектурном цикле

| Роль | Вклад |
|------|--------|
| **Архитектор (10)** | Канон, ADR, review модулей, сквозные контракты |
| **Разработчик (03)** | Реализация, тесты, предложения по trade-off |
| **Директор (01)** | Scope, срок vs качество, approve крупных ставок |
| **Сисадмин (04)** | Runtime, деплой, observability stack |
| **Тестировщик (09)** | Поведение по ТЗ; архитектурные баги — через дефект + архитектор |
| **Дизайнер (08)** | UI-токены; архитектор — не блокирует пиксели, но — shell/zones |
| **Юрист (07)** | ПД и акцепты; архитектор — техническая реализация следа |

## Ритм

- **Старт модуля:** разработчик → архитектор: «граница и api-http черновик» (30 мин async или созвон).
- **Перед merge:** чеклист; спор → ADR proposed → accepted за 48 ч или эскалация директору.
- **Раз в 2–4 недели:** архитектурный обзор (30 мин): техдолг, расхождение docs/code, пункты backlog §6 в «Что фиксировать».

## Эскалации

| Ситуация | Кому |
|----------|------|
| Breaking API без миграционного плана | Директор + разработчик |
| Нужен второй сервис / очередь / БД | Директор; сисадмин на оценку ops |
| Конфликт скорости MVP vs границы модулей | Директор |
| Инцидент из-за связности модулей | Архитектор → ADR + fix plan |

## Чат роли

См. `Команда/Чаты по ролям.md` — префикс **`[10 Архитектор]`**.

Шаблон:

`Работаем как архитектор приложения. Папка Команда/10-Архитектор/. Цель: ADR для … / review M2 / ревизия offline.`

## Совмещение с разработчиком

Если один человек — в чате явно переключать режим: **«сейчас архитектурный review»**, не смешивать с «быстро допилить фичу» в одном проходе без чеклиста.
