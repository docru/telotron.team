# Архитектор приложения (10)

Роль **архитектурной целостности** продукта Telotron: границы модулей, контракты, сквозные решения (зоны, offline, API), чтобы сопровождение не превращалось в «монолит без правил».

На текущем этапе роль **может совмещаться** с опытным разработчиком; отдельный штат — когда объём модулей и параллельных фич мешает успевать и кодить, и держать канон.
