# Команда — текущий этап

## Цель
Зафиксировать минимально необходимую команду на текущем этапе и распределить зоны ответственности без лишней бюрократии.

## Состав специалистов
0. Священник
1. Директор
2. Маркетолог
3. Разработчик
4. Сисадмин
5. Бухгалтер
6. Поддержка
7. Юрист
8. Дизайнер
9. Тестировщик
10. Архитектор приложения

## Продуктовые ограничения (канон)

- **SMS не используем** (дорого): OTP и уведомления — **MAX**, **e-mail**, **Telegram**; см. [_telotron.ru/docs/Бизнес-требования/02-модули/auth/регистрация-и-вход-max-passkey.md](../_telotron.ru/docs/Бизнес-требования/02-модули/auth/регистрация-и-вход-max-passkey.md) § «Решение: SMS не используем».
- **Роскомнадзор:** без **поданного уведомления** об операторе ПДн — **нет** периода I и подключения реальных пользователей; см. [уведомление-роскомнадзор-оператор-пд](../_telotron.ru/docs/Бизнес-требования/04-комплаенс-и-аудит/уведомление-роскомнадзор-оператор-пд.md).

## Принцип
- На текущем этапе роли могут совмещаться одним человеком.
- Отдельные менеджеры продаж не вводятся.
- Аккаунт-менеджеры рассматриваются позже, при росте платящих подключений.
