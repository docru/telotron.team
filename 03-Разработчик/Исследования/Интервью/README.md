# Расшифровка интервью (Docker)

## Что нужно

- Установленные **Docker** и **Docker Compose**.

## Подготовка (один раз)

1. Положи аудиофайл в папку **`Звук/`** (mp3, wav, amr и т.п.).

2. В терминале:

```bash
cd /путь/к/папке/Интервью   # где лежат docker-compose.yml и transcribe.sh
chmod +x transcribe.sh
docker compose build
```

Пример у тебя на диске:

`~/Педагогика/ПРОЕКТЫ/Приложение для фитнес-тренеров и диетологов/Интервью`

Первая сборка может занять несколько минут.

## Запуск

```bash
./transcribe.sh ИМЯ_ФАЙЛА_ИЗ_ЗВУК
```

Пример:

```bash
./transcribe.sh interview_01.mp3
```

Модель по умолчанию — **`small`**. Другая модель вторым аргументом:

```bash
./transcribe.sh interview_01.mp3 medium
```

## Результат

- В папке **`Текст/`** появится файл **`ИМЯ.txt`** (то же имя, что у аудио, без расширения).
- Сохраняется только **`.txt`**; файлы `.vtt`, `.srt`, `.json`, `.tsv` с тем же именем удаляются.

## Если менялся `transcribe.py` или `Dockerfile`

Пересобери образ:

```bash
docker compose build
```

## Кэш моделей

Модели хранятся в Docker-томе **`whisper_cache`** (не в домашней папке). Чтобы скачать модель заново с нуля:

```bash
docker volume rm interview-whisper_whisper_cache
```

(потом при следующем запуске снова пойдёт загрузка).
