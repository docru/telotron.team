# Функционал: модули, группировка и оценка сроков

Исходный список: `Функционал.md`.

**Кто оценивает:** один программист **высокой квалификации** (full‑stack: бэкенд + фронт + базовая инфраструктура + продакшен‑минимум), без отдельного дизайнера и без отдельного QA (ручное тестирование сам).

**Единицы:** «нед» = одна календарная неделя при ~40 ч/нед на задачу.

**Сложность:** S (до ~1 нед) · M (1–3 нед) · L (3–6 нед) · XL (2–4 мес и более).

**Важно:** оценки **не суммируются линейно** при параллельной работе; ниже — порядок величины для **последовательной** реализации модуля «с нуля до продакшен‑качества». Интеграции с банками и ФНС сильно зависят от документации и доступов.

---

## 1) Роли и клиенты платформы

| Роль | Суть в продукте |
|------|-----------------|
| Клиент | Личный кабинет: планы, трекинг, оплаты, связь со специалистом |
| Специалист | Рабочее место: CRM, журнал, планы, трекинг клиента, оплаты, группы |
| Маркетолог | Каналы роста и партнёрка (инструменты роста по мере необходимости) |
| Поддержка | Сопровождение и удержание после 500-1000 платящих |
| Руководитель | Дашборд компании |

**Модуль M0. Платформа и роли**

- Единая авторизация, роли и права (RBAC), разделение «клиент / специалист / сотрудник компании».
- Один **веб-клиент** (PWA) с разными разделами **или** два клиентских фронта на одном API — решение влияет на срок на 2–4 нед.

**Оценка:** **L**, **4–6 нед** (если один фронт + роли). **+2–4 нед**, если два отдельных клиентских UX с общим API.

---

## 2) Карта модулей (группировка по функционалу)

### M1. CRM и карточка клиента

- Контакты, заметки, фильтры, история взаимодействий.
- Пересечение с контуром маркетинга/поддержки — общие сущности «лид / клиент / специалист».

**Оценка:** **L**, **4–6 нед** (MVP: клиенты + заметки + теги; без тяжёлой воронки).

---

### M2. Журнал и календарь

- Электронный журнал занятий, неделя/месяц, перетаскивание (как в исходнике).

**Оценка:** **M–L**, **2–5 нед** (простой календарь меньше; drag‑and‑drop + повторяющиеся слоты + конфликты — больше).

---

### M3. Коммуникации: чат

- Чат с клиентами, привязка сообщений к упражнению / плану.

**Оценка:** **L**, **4–7 нед** (realtime, вложения, привязка к сущностям, модерация). Упрощённый «чат без привязок» — **M**, **2–3 нед**.

---

### M4. Уведомления и напоминания

- Напоминания о встрече, о трекерах; дублирование в мессенджеры (интеграция отдельным подмодулем).

**Оценка:** **M**, **2–4 нед** (внутренние напоминания). **+M**, **+2–3 нед** на стабильный канал (например, Telegram‑бот) как MVP вне push.

---

### M5. Планы тренировок и питания (контент)

- Планы, готовые шаблоны, рецепты с КБЖУ, статьи / микроконтент для клиента.

**Оценка:** **L**, **4–6 нед** (редактор контента + выдача клиенту + версии). «Статьи» как простые страницы — **S–M**, **1–2 нед** поверх редактора.

---

### M6. Конструкторы шаблонов

- Конструктор шаблонов питания и тренировок (в исходнике отдельно).

**Оценка:** **XL**, **2–4 мес** каждый, если делать богатый UI и правила. Урезанный конструктор («шаблон из блоков + копирование») — **L**, **5–8 нед** на оба направления в одном стиле.

---

### M7. Трекинг (клиент + вид специалиста)

- Питание, тренировки, вода, сон, активность, замеры; отклонения от плана; фото еды; мотиваторы достижений.

**Оценка:** **XL**, **2–4 мес** на «всё из списка» с аналитикой отклонений. MVP «вес + шаги + вода + дневник приёма пищи» — **L**, **5–8 нед**.

---

### M8. Медиа: видео тренировок

- Загрузка или ссылки на видеохостинги.

**Оценка:** **M–L**, **2–5 нед** (embed быстрее; своя загрузка + хранение + транскодинг — дольше и дороже в эксплуатации).

---

### M9. Оплаты и «Мой налог»

- Приём оплаты через платформу, запись в «Мой налог», идея удержания/налога.

**Оценка:** **L**, **4–7 нед** (платёжный провайдер + webhooks + статусы). **Мой налог:** **M–L**, **2–6 нед** в зависимости от выбранного сценария (только чеки vs глубже) и стабильности API/документации. Юридическая корректность сценариев — вне оценки кода.

---

### M10. Аналитика и «рост бизнеса»

- Рекомендации по удержанию, привлечение, акции, расчёт бизнес‑показателей, автоматический расчёт КБЖУ/тоннаж/ССЗ и др.

**Оценка:**  
- Дашборды по **фактам** (выручка, посещаемость, воронка вручную заведённая): **L**, **4–6 нед**.  
- «Рекомендации» как **правила** (if‑then по метрикам): **L**, **+3–5 нед**.  
- «Умные» рекомендации как **ML** — отдельный продукт; **XL**, не закладывать в первый релиз без исследования.

---

### M11. Групповой режим

- Оплаты, оповещения, чат группы (из исходника).

**Оценка:** **L**, **4–7 нед** поверх M2+M3+M4+M9 (модель группы, роли в группе, рассылки).

---

### M12. Персонализация

- Фото, цвета интерфейса.

**Оценка:** **S–M**, **1–2 нед**.

---

### M13. Визуализация прогресса

- Графики динамики для клиента и специалиста.

**Оценка:** **M**, **2–4 нед** на связке с трекером (данные должны быть чистыми).

---

### M14. Обратная связь

- Сбор отзывов об удобстве, багрепорты.

**Оценка:** **S**, **3–7 дн**.

---

### M15. Интеграции (подключаемые блоки)

| Интеграция | Оценка |
|------------|--------|
| Платежи (ЮKassa и др.) | см. M9 |
| Мой налог | см. M9 |
| Мессенджеры (дублирование) | **M**, **2–4 нед** на первый канал |
| Пуш‑сервис (PWA ограничения на iOS) | **M**, **2–4 нед** (MVP), дальше — политика каналов |
| «Коллаж / маркетинговые сервисы» | **M–L** каждая, после уточнения API |

---

### M16. Панели маркетолога / поддержки / руководителя

- CRM, финансы, использование инструментов, чат со специалистом, показатели компании.

**Оценка:** **L**, **5–8 нед** после M1 (много отчётов = много уточнений). Сильно сокращается, если **не строить вторую CRM**, а дать **экспорт/просмотр** метрик специалиста.

---

## 3) Шаблоны контента (не код, но блокируют продукт)

- Рецепты, наборы шаблонов питания, набор упражнений, планы тренировок.

**Оценка по разработке:** **S**, **формально 0** (контент готовят не разработчиком).  
**По продукту:** без минимального набора MVP «пустой» — заложить **2–4 нед** работы методиста параллельно.

---

## 4) Рекомендуемые фазы (для одного сильного разработчика)

Ориентир «живой» продукт для специалиста + клиента, без тяжёлой корпоративной части:

| Фаза | Состав | Ориентир по времени |
|------|--------|---------------------|
| **Фаза 0** | M0 + M14 + деплой | **4–8 нед** |
| **Фаза 1 (MVP ядро)** | M1 + M2 + M4 (минимум) + M5 (чтение планов) + M13 (базово) | **+8–14 нед** |
| **Фаза 2** | M3 + M7 (урезанный трекинг) + M8 (embed) | **+10–16 нед** |
| **Фаза 3** | M9 + M15 (платежи + один мессенджер) | **+6–12 нед** |
| **Фаза 4** | M6 (урезанные конструкторы) | **+8–16 нед** |
| **Фаза 5** | M11 группы + M10 правила | **+8–14 нед** |
| **Фаза 6** | M16 корпоративные роли | **+5–10 нед** или упростить |

**Итого до «богатого» продукта из исходника:** порядка **1,5–3+ года** одним разработчиком при последовательной подаче. **MVP для интервью и пилотов** реалистичнее укладывать в **4–6 месяцев** при жёстком срезе фич.

## 6) Риски, завышающие сроки (для планирования)

- Реалтай‑чат и вложения.
- Конструкторы с «бесконечной» гибкостью.
- Видео на своём хранилище.
- Налоговые и платёжные сценарии без чёткого юридического сценария.
- Ожидание «умных» рекомендаций без формализованных правил.

Если нужно, следующим шагом могу **разметить каждый пункт из `Функционал.md` галочками** «фаза 1 / 2 / отложено» под твой пилот в Краснодаре.
