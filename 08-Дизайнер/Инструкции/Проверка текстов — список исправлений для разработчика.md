# Проверка текстов — список исправлений для разработчика

**Статус:** ревью дизайнера (май 2026), **без правок в коде** — вносит разработчик.  
**Связь:** [Решения совещания — UI](Решения%20совещания%20—%20требования%20UI.md) · [Спецификация экранов MVP](Спецификация%20экранов%20MVP.md) · [Система стилей](Система%20стилей%20—%20Pro%20и%20Client%20(MVP).md) · `resources/ts/shared/telotron-theme.ts` (`voiceCopy`)

---

## 1. Задачи текстов (как у дизайна)

Тексты должны обслуживать **те же цели**, что визуал:

| Зона | Задача UI (директор + маркетолог + дизайн) | Задача текстов |
|------|--------------------------------------------|----------------|
| **Pro** | Ежедневный **инструмент** тренера: клиенты, календарь, инвайт; без «витрины» | Быстро понять **что сделать**; деловой тон; без жаргона продукта/разработки |
| **Client** | **Подарок от тренера**, не реклама платформы; один сценарий на экран | Простой язык **среднего человека**; «ты»; офлайн = «позже запишусь», не «ошибка» |

## 2. Целевая аудитория (маркетолог + интервью MVP)

**Источники:** [MVP — сверка](../../01-Директор/Инструкции/MVP%20—%20сверка%20маркетолог%20и%20разработчик.md) · интервью Катя, Павел, Максим (`Команда/03-Разработчик/Исследования/Интервью/Обобщение интервью/`).

### 2.1. Pro — кто читает экран

**Сегмент MVP (условно 3 голоса):** практикующий **тренер / нутрициолог / диетолог**, чаще **solo**, **10–40 клиентов**, онлайн или гибрид.

| Черта | Как писать | Как не писать |
|-------|------------|---------------|
| Время дорого | Короткие заголовки, глагол в CTA | Длинные абзацы, «возможности платформы» |
| Стек сейчас: Telegram, таблицы, YCLIENTS/InSkill | Слова **клиенты, занятие, расписание, группа, план, ссылка** | CRM, workspace, зона, API, токен, Passkey в основном UI |
| Хочет «всё в одном месте» | «Клиенты и занятия в одном приложении» (смысл, не слоган на каждом экране) | Три разных «приложения» в подписях |
| Time-to-value &lt; 10 мин | На пустом списке: **пригласить → ссылка**; в профиле — без лекций | Технические условия активации профиля абзацем |
| Не все — студии с 30 группами | Группы — понятно, но **инвайт клиента** важнее | Перегруз маркетингом, партнёрская ссылка = главная |

**Тон Pro:** обращение **«вы»** (ошибки, подсказки, баннеры). CTA без местоимения: «Пригласить клиента», «Создать занятие».

### 2.2. Client — кто читает экран

**Не тренер.** Человек, которого **пригласил свой тренер**; часто не power-user фитнес-приложений.

| Черта | Как писать | Как не писать |
|-------|------------|---------------|
| Не платит за сервис | «От тренера», «ваше занятие» | «Подписка», «тариф», реклама Телотрона |
| Один экран — одна задача | «Записаться», «Подтвердить», «Сегодня» | Меню из 8 разделов с одинаковым весом |
| Средний уровень цифровой грамотности | «Нет интернета», «Слот занят» | Синхронизация, конфликт, outbox, WebAuthn |
| Мотивация — поддержка | «Привет, {имя}», прогресс цифрой | Давление, стоковые «мотивашки» |

**Тон Client:** **«ты»** в сообщениях и подсказках. CTA: «Записаться», «Подтвердить» (без «ты» в кнопке).

**Исключения (везде «вы»):** регистрация/юридические блоки, публичные страницы, Admin.

## 3. Общие правила (обе зоны)

1. **Один термин — одно слово** в продукте (см. таблицу §4).
2. **Нет ID в UI:** не показывать `Клиент #12`, `Тренер #5` — только имя или «Тренер» / ФИО из API.
3. **Нет отсылок к ТЗ/API/разработке** в пользовательских сообщениях.
4. **Ошибка = что случилось + что сделать** (одно–два предложения).
5. **Офлайн Pro:** изменения расписания только онлайн. **Офлайн Client:** запись при сети; просмотр можно.
6. Централизовать микрокопи в **`voiceCopy`** (`telotron-theme.ts`) + по возможности не дублировать строки в Vue.

## 4. Словарь терминов (канон)

| Концепт | Pro | Client | Убрать / не использовать в UI |
|---------|-----|--------|-------------------------------|
| Приложение | Телотрон **Pro** (в шапке) | Телотрон **Client** | «зона Pro/Client», workspace |
| Нижняя навигация | **Клиенты** · **Календарь** · **Ещё** | **Главная** · **Календарь** · **Планы** | «Записи» (если tab = Календарь) |
| Расписание тренера | **Календарь** (tab и заголовок) | — | «Расписание» в tab (допустимо только в «Ещё» как синоним с пояснением) |
| Запись клиента | — | **Записаться** / **Календарь** | Заголовок страницы «Записаться» при tab «Календарь» |
| Занятие | **Занятие** | **Тренировка** / **Занятие** (одно выбрать; рекомендация: **занятие** везде) | «Сессия», session_type в UI |
| Режимы календаря | **Месяц** · **Неделя** · **День** · **Список** | **Список** (agenda) основной | Month, Week, **Лента** (если путает с соцсетью) |
| Passkey | **Вход по отпечатку / Face ID** (подзаголовок) | то же | Passkey, WebAuthn — только в «Подробнее» или FAQ |
| Синхронизация | **Сохраняем…** / **Сохранено** | **Сохраняем…** / **Не отправилось — проверь сеть** | «Несинхронизированные данные», «Конфликт» без пояснения |

## 5. Канонические фразы (`voiceCopy` — расширить)

Текущий `voiceCopy` — минимум. Добавить/заменить по таблице:

### Pro (`voiceCopy.pro`)

| Ключ | Текст |
|------|-------|
| `offlineSchedule` | Изменения расписания доступны только при подключении к интернету |
| `offlineNetwork` | Нет интернета. Проверьте подключение |
| `slotConflict` | Это время уже занято. Выберите другое |
| `emptyClients` | Пока нет клиентов |
| `emptyClientsLead` | Отправьте ссылку — клиент зарегистрируется сам |
| `inviteCta` | Пригласить клиента |
| `clientsActiveCount` | `{n} в базе` или `{n} клиентов` (не «активных» без определения) |
| `listNextLesson` | Следующее: {дата время}` или `Нет занятий в расписании` |
| `copySuccess` | Ссылка скопирована |
| `cancelReasonRequired` | Напишите, почему отменяете занятие |
| `cancelSuccess` | Занятие отменено |
| `loadErrorList` | Не удалось загрузить список. Попробуйте ещё раз |
| `featureGateGroups` | Групповые занятия доступны на тарифе Профи |

### Client (`voiceCopy.client`)

| Ключ | Текст |
|------|-------|
| `offlineBooking` | Записаться можно при подключении к интернету |
| `offlineNetwork` | Нет интернета |
| `slotConflict` | Это время уже занято. Выбери другое |
| `greeting` | Привет, {name} |
| `bookingSuccess` | Ты записан на {дата, время} |
| `confirmLesson` | Подтверди занятие {когда} |
| `emptyPlans` | Тренер пока не прислал планов |
| `emptySlots` | В этот день нет свободного времени |
| `slotFree` | Свободно |
| `syncPending` | Сохраняем… |
| `syncFailed` | Не удалось отправить. Проверь интернет |

## 9. Чеклист приёмки текстов (QA)

- [ ] Pro: везде «вы» в сообщениях; Client: «ты» (кроме auth/юридического).
- [ ] Нет слов: зона, API, WebAuthn, CRM, #id, ТЗ, workspace в пользовательском UI.
- [ ] Tab «Календарь» и заголовок экрана записи Client согласованы.
- [ ] Pro: «Пригласить клиента» — главный CTA; партнёрская ссылка не выглядит как клиентская.
- [ ] Офлайн: Pro — не изменить расписание; Client — записаться позже.
- [ ] Ошибка 409: понятна без кода ответа.
- [ ] Пустые состояния: что делать дальше (одна кнопка).
- [ ] Тренер из интервью поймёт экран за 3 секунды без инструкции.
- [ ] Нетехнический клиент поймёт главную и запись без инструкции.
