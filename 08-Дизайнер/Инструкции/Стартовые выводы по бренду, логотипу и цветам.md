# Стартовые выводы по бренду, логотипу и цветам

## 1) Базовое решение по бренду

- Для проекта используем **один общий бренд**: одно название и один логотип.
- Утвержденное название бренда: **Телотрон**.
- **Мобильные приложения** (и веб-клиенты) именуем по ролям:
  - `Телотрон Pro` — для специалистов (тренеры и диетологи).
  - `Телотрон Client` — для клиентов.
- Визуальный вектор у интерфейсов разный, но система идентичности единая.

## 2) Почему это правильно

- Сохраняется целостность бренда и узнаваемость.
- Пользователь не путается в экосистеме (понятно, что это один сервис).
- Можно гибко адаптировать UX/UI под разные сценарии работы:
  - `Pro`: управление, аналитика, контроль.
  - `Client`: простота, поддержка, мотивация.

## 3) Логотип (единый для зон Pro и Client)

### Принцип

- Один знак и одно начертание названия.
- Допускаются две цветовые версии логотипа (для `Pro` и `Client`), но форма не меняется.

### Идея знака

- Комбинация смыслов: **прогресс + здоровье + сопровождение**.
- Вариант направления: монограмма + элемент пульса/листа/галочки.

## 4) Цветовая система (две ветки)

**Канон для MVP (токены, HEX, правила):** [Токены цветов — Pro и Client](Токены%20цветов%20—%20Pro%20и%20Client.md).

Ниже — стартовое направление; где расходится с каноном токенов, действует документ токенов.

### 4.1 Для специалистов (`Pro`)

Характер: структурность, экспертность, надежность.

- Основной: `#1D4ED8` (синий) → токен `primary`
- Текст: `#0F172A` (графит) → `foreground`
- Предупреждения: `#D97706` → `warning` (не CTA; ранний черновик `#F59E0B` — только смысл warning)
- Фон: `#F8FAFC` → `background`

### 4.2 Для клиентов (`Client`)

Характер: поддержка, прогресс, легкость.

- Основной CTA: `#16A34A` (зелёный) → `primary` (в каноне; не `#22C55E` для кнопок)
- Фон: `#FFFFFF` / мягкий `#F0FDF4` → `background` / `primary-subtle`
- Бирюза `#06B6D4` и розовый `#FB7185` из раннего черновика — **не** в системных токенах MVP (см. канон)

## 5) Типографика и стиль

- Шрифтовая система общая для обеих веток (Pro и Client).
- Отличие по стилю:
  - `Pro`: плотнее сетка, больше табличных и аналитических блоков.
  - `Client`: больше воздуха, крупнее ключевые действия, меньше когнитивной нагрузки.

## 6) Рабочая схема именования

Формат:

- `Телотрон Pro`
- `Телотрон Client`

Пример:

- `Телотрон Pro`
- `Телотрон Client`

## 7) Что зафиксировать следующим шагом

1. Финальное имя бренда утверждено: `Телотрон`.
2. Утвердить концепт логотипа (1 знак + 2 цветовые версии).
3. ~~Описать токены UI-цветов~~ — сделано: [Токены цветов — Pro и Client](Токены%20цветов%20—%20Pro%20и%20Client.md).
4. Подготовить 2 референс-экрана:
   - Дашборд специалиста (`Pro`).
   - Главный экран клиента (`Client`).
5. Привести код Client к `theme-client` (зелёный `theme_color` и `primary`, см. §7 канона токенов).
