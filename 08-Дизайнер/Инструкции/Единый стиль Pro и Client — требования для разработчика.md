# Единый стиль Pro и Client — требования для разработчика

**Статус:** краткое введение; **полный канон** — [Система стилей — Pro и Client (MVP)](Система%20стилей%20—%20Pro%20и%20Client%20(MVP).md)  
**Аудитория:** разработчик (вёрстка, `resources/css`, shadcn-vue)  
**Связано:** [Система стилей (MVP)](Система%20стилей%20—%20Pro%20и%20Client%20(MVP).md) · [Токены цветов](Токены%20цветов%20—%20Pro%20и%20Client.md) · [Решения совещания — UI](Решения%20совещания%20—%20требования%20UI.md)

## 1. Принцип: один Телотрон, две темы

Пользователь должен узнавать **один продукт** (логотип, формы кнопок, карточки, навигация, иконки), но **не путать роли**: Pro — работа, Client — сопровождение.

| Слой | Общий для Pro и Client | Разный по зоне |
|------|------------------------|----------------|
| Компоненты | Кнопка, Card, Sheet, Toast, Input, Tab bar, Avatar (инициалы), Skeleton | — |
| Геометрия | Радиусы, тени, отступы (шкала 4px), высота tab bar | — |
| Шрифт | **Geist** (уже в `app.css`) | Размер body / плотность |
| Иконки | **Lucide**, один stroke-width | Акцентный цвет иконки |
| Layout | Карточный UI, нижняя навигация, bottom sheet | Состав вкладок |
| Цвет | Нейтрали (фон, текст, border, destructive) | **Primary** и акценты |
| Копирайт | Короткие CTA без «ты/вы» | Тон ошибок и подсказок |

**Не делаем:** два разных «приложения» (Sportup для Pro + FITNEX для Client).  
**Делаем:** одна дизайн-система + классы **`theme-pro`** / **`theme-client`** на `<html>` или корневом layout зоны.

---

## 2. Референсы (как читать пины)

| Пин | Роль в системе |
|-----|----------------|
| CRM mobile (синий дашборд) | **Каркас для обеих зон:** карточки, KPI-формат, quick actions, tab bar, типографика цифр |
| Sportup | **Паттерны экранов:** календарь (полоса дат), списки, подтверждение записи — те же компоненты, другой primary |
| FITNEX | **Только смысл Client:** блок «Сегодня», простой график веса — **не** оранжевый primary и не meal-planner |

---

## 3. Общие токены (не менять между зонами)

Имена — как в shadcn / `app.css`. Значения нейтрали **одинаковые** в Pro и Client.

| Токен | Значение (MVP) | Примечание |
|-------|----------------|------------|
| `--font-sans` | Geist | Уже подключён |
| `--radius` | `0.75rem` (12px) | Кнопки pill: `rounded-full` для primary CTA |
| `--radius-lg` | `1rem` (16px) | Карточки |
| `--background` | Pro: `#F8FAFC`, Client: `#FFFFFF` | См. §4 — допустимо различие surface |
| `--foreground` | `#0F172A` (slate-900) | Общий текст |
| `--card` | `#FFFFFF` | |
| `--border` | `#E2E8F0` (slate-200) | |
| `--muted` | `#F1F5F9` | Фон вторичных блоков |
| `--muted-foreground` | `#64748B` | Подписи |
| `--destructive` | без смены по зоне | Ошибки, отмена |
| Тень карточки | `0 1px 3px rgb(15 23 42 / 0.08)` | Одна формула |

**Компоненты (один код, один вид формы):**

- Primary button: pill, высота Pro **40px**, Client **44px** (через `--btn-height` или utility класс зоны).
- Secondary: outline, тот же radius.
- Card: белая, padding `16px`, заголовок semibold.
- Sheet / Toast: одни и те же; меняется только цвет primary в accent.
- Avatar: круг, **инициалы**, без обязательного фото.
- Offline-banner: общий компонент; Client дополнительно стиль слота `--offline-slot-opacity: 0.4`.

## 4. Темы зон (меняется только «акцентная ветка»)

Подключение: на layout Pro — `class="theme-pro"`, Client — `class="theme-client"` (на `html` или `#app`).

### 4.1. `theme-pro`

| Токен | HEX | Использование |
|-------|-----|----------------|
| `--primary` | `#1D4ED8` | CTA, active tab, ссылки |
| `--primary-foreground` | `#FFFFFF` | Текст на primary |
| `--accent` | `#F59E0B` | Предупреждения, контрольные точки (не второй primary) |
| `--background` | `#F8FAFC` | Фон приложения |
| Плотность | `font-size: 14px` (body), списки compact | |

### 4.2. `theme-client`

| Токен | HEX | Использование |
|-------|-----|----------------|
| `--primary` | `#16A34A` | CTA, active tab (канон; не `#22C55E`) |
| `--primary-foreground` | `#FFFFFF` | |
| `--primary-subtle` | `#F0FDF4` | Фон блока «Сегодня» |
| `--background` | `#FFFFFF` | |
| Плотность | `font-size: 16px` (body), кнопки min **44px** | |

### 4.3. Пример CSS (ориентир для `app.css`)

```css
:root {
  /* общие нейтрали — см. §3 */
}

.theme-pro {
  --primary: #1d4ed8;
  --primary-foreground: #ffffff;
  --accent: #f59e0b;
  --background: #f8fafc;
  --telotron-zone-density: compact; /* опционально: utility */
}

.theme-client {
  --primary: #16a34a;
  --primary-foreground: #ffffff;
  --primary-subtle: #f0fdf4;
  --background: #ffffff;
  --calendar-slot-offline-opacity: 0.4;
}
```

Конвертация в `oklch` — по желанию, главное **стабильные HEX** в макете и в PR.

## 5. Общие паттерны экранов (одинаковая «мебель»)

### 5.1. Shell (обе зоны)

```
┌─────────────────────────────┐
│ [лого]          [🔔] [👤]  │  ← одна высота header
├─────────────────────────────┤
│  контент (карточки/списки)  │
├─────────────────────────────┤
│  Tab · Tab · Tab · (Tab)    │  ← bottom nav, 3–4 пункта
└─────────────────────────────┘
```

- Viewport ориентир **~430px**, safe-area снизу.
- Логотип: **один знак Телотрон**; подпись «Pro» / «Client» рядом или в manifest, не другой графический стиль.

### 5.2. Карточка метрики (CRM-паттерн, обе зоны)

Один Vue-компонент, например `MetricCard`:

- Заголовок muted, small  
- Число крупное (24–28px, semibold)  
- Опционально бейдж тренда (↑ зелёный / ↓ красный) — **те же классы** в Pro и Client  

**Pro:** «Занятий сегодня», «Ожидают подтверждения», «Активных клиентов».  
**Client:** «Занятий на неделе», «Записей в дневнике», «Вес за месяц» — те же карточки, другой primary у иконки.

### 5.3. Quick Actions (сетка 2×2)

Один компонент `QuickActionGrid`: светло-голубой/светло-зелёный фон иконки = `primary/10` от **текущей** темы.

| Pro | Client |
|-----|--------|
| Пригласить клиента | Записаться |
| Календарь | Мои планы |
| … | Дневник (M7-lite) |

### 5.4. Списки и календарь

- Строка списка: avatar + title + subtitle + chevron — **один** `ListRow`.
- Календарь: полоса дат + список/agenda (Sportup) — **один** `CalendarStrip` + разный default view (Pro: month, Client: agenda).
- Запись слота: **один** `BookingSheet`; offline — opacity + иконка сети только Client.

### 5.5. Состояния (общие)

| Состояние | Поведение |
|-----------|-----------|
| Loading | Skeleton тех же карточек |
| Empty | Абстрактная иллюстрация (3–4 варианта на продукт), без стоковых фото |
| Error / 409 | Toast + текст; Pro «вы», Client «ты» |
| Gate тарифа | Один компонент `FeatureGate` (иконка замка + copy), не серый «сломанный» UI |

## 6. Чем зоны **сознательно** отличаются (не унифицировать)

| Аспект | Pro | Client |
|--------|-----|--------|
| Primary | Синий | Зелёный |
| Плотность | Плотнее | Больше воздуха |
| Центр UX | Список клиентов + sticky «Пригласить» | Главная: Сегодня / Занятия / Планы |
| Тон | «Вы» | «Ты» |
| Дашборд KPI | До 3 карточек, не карусель ради KPI | Блок «Сегодня» важнее графиков |
| Офлайн | Баннер «мутации только онлайн» | Тусклые слоты + «запись при сети» |
| Tab bar | Клиенты · Календарь · Ещё | Главная · Календарь · Планы · (Профиль) |

Пользователь видит: **те же кнопки и карточки**, **другой цвет и тон** — «один Телотрон, моя роль понятна».

## 7. Что править в коде (чеклист)

1. **`resources/css/app.css`** — вынести нейтрали в `:root`, добавить `.theme-pro` / `.theme-client`.
2. **Layout Pro / Client** — класс темы на корень (`pro.layout`, `client.layout` или Vue root).
3. **Не дублировать** отдельные Button/Card для зон — только CSS-переменные и 1–2 utility (`text-base` vs `text-sm` по зоне).
4. **Общие компоненты** в `resources/ts` (или `components/ui`): `MetricCard`, `QuickActionGrid`, `ListRow`, `OfflineBanner`, `FeatureGate`.
5. **Иконки** — Lucide везде; цвет `text-primary` от темы.
6. **Не внедрять** из референсов: оранжевый FITNEX primary, marketplace Sportup, sales-лексику CRM (leads/deals), стоковые фото, кольца KPI на каждом экране.

---

## 8. Приоритет экранов для стилизации (MVP)

Один проход «shell + темы», затем:

**Pro:** список клиентов → карточка клиента → календарь (неделя) → инвайты.  
**Client:** главная (online + offline в одном макете) → календарь agenda → онбординг (3 шага).

Остальные экраны §14 наследуют shell без отдельного «второго дизайна».

---

## 9. Критерий приёмки (для тестировщика)

- [ ] Соседние экраны Pro и Client выглядят **родственно** (шрифт, формы, карточки, nav).
- [ ] Primary **синий только на Pro**, **зелёный только на Client** (нет перекрёста).
- [ ] Один и тот же Toast/Sheet/кнопка в обеих зонах — **та же геометрия**.
- [ ] Client не ощущается CRM; Pro не ощущается фитнес-трекером с едой/шагами.
- [ ] Нет стоковых фото людей на пилотных экранах.
