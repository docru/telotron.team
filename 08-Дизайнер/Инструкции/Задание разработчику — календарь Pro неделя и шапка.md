# Задание разработчику: календарь Pro — вид «Неделя» и шапка страницы

**От:** дизайнер (08)  
**Статус:** к передаче в разработку  
**Связь:** [Спецификация экранов MVP §3](Спецификация%20экранов%20MVP.md) · [Система стилей §7.11–7.12](Система%20стилей%20—%20Pro%20и%20Client%20(MVP).md) · [календарь-и-запись](../../../_telotron.ru/docs/Бизнес-требования/02-модули/m2-календарь/календарь-и-запись.md)

**Контекст от заказчика (2026-05-21):** на скриншоте вид «Неделя» с горизонтальной прокруткой неудобен на 430px; подзаголовок про «только онлайн» кажется лишним; кнопка «Новое занятие» на отдельной строке съедает высоту — нужно компактнее, но CTA остаётся заметным.

---

## 1. Проблема

### 1.1. Вид «Неделя»

| Сейчас | Почему плохо |
|--------|----------------|
| `TelotronCalendar.vue`, `mode === 'week'`: `overflow-x-auto` + `flex min-w-[640px]` | На PWA **430px** семь колонок не помещаются — горизонтальный скролл, мелкий текст, лишний жест |
| 7 узких колонок в один ряд | Сложно сравнить дни и тапнуть занятие |

### 1.2. Шапка страницы «Календарь» (Pro)

Текущая структура `WorkspaceSchedulePage.vue`:

```
h1 «Календарь»
p  «Занятия по дням. Менять расписание можно только онлайн»   ← voiceCopy.pro.scheduleLead
[полная строка] Button pill «Новое занятие»
TelotronCalendar (внутри: переключатель вида + навигация дат + сетка)
```

| Элемент | Замечание |
|---------|-----------|
| **Подзаголовок** `scheduleLead` | Дублирует идею офлайна: при **онлайн** строка не даёт пользы; при **офлайн** уже есть `TelotronOfflineBanner` в shell (`TelotronAppShell`) и жёлтый блок внутри календаря (`readonlySlots` → `calendarOffline`) |
| **Кнопка «Новое занятие»** | `justify-end` на всю ширину — **отдельная строка** между заголовком и календарём (~48px + отступы) |
| **Итого** | До сетки недели пользователь прокручивает ~3 блока текста/кнопок |

---

## 2. Цель

1. **Неделя:** все **7 дней** видны на экране **без горизонтальной прокрутки** (ширина колонки приложения `max-w-[430px]`).
2. **Шапка:** убрать избыточный copy про онлайн; **«Новое занятие»** остаётся **primary CTA**, но занимает **меньше вертикали**.
3. Поведение офлайн **не ослабить**: без сети — нет создания/редактирования; подсказка офлайн — **один** канал (banner shell), не три текста.

---

## 3. Решение

### 3.1. Вид «Неделя» — сетка 2×4 (7 дней)

Заменить горизонтальную полосу на **CSS grid**:

| Параметр | Значение |
|----------|----------|
| Колонки | **2** (`grid-cols-2`) |
| Ряды | **4** (8 ячеек, **7** с контентом дня) |
| Порядок дней | **Пн → Вс** (как сейчас `startOfWeekMonday`) |
| 8-я ячейка | Пустая **или** невидимый spacer (не показывать «мусорный» день) |

**Схема (пример недели 18–24 мая):**

```
┌─────────────┬─────────────┐
│ Пн 18.5     │ Вт 19.5     │  ряд 1
├─────────────┼─────────────┤
│ Ср 20.5     │ Чт 21.5     │  ряд 2
├─────────────┼─────────────┤
│ Пт 22.5     │ Сб 23.5     │  ряд 3
├─────────────┼─────────────┤
│ Вс 24.5     │  (пусто)    │  ряд 4
└─────────────┴─────────────┘
```

**Альтернатива (допустима при приёмке):** воскресенье на всю ширину (`col-span-2`) в 4-м ряду — если визуально лучше; согласовать с §5.

**Ячейка дня** (сохранить логику из текущей колонки):

- Шапка: день недели + дата (клик → переход в «День», как `onDayClick`).
- Тело: список занятий (`entriesOnDay`), скролл **внутри ячейки** по вертикали при переполнении (`max-h` ~ `10–12rem`, не на всю страницу).
- **Сегодня:** акцент `text-primary` / `ring` как сейчас.
- Карточка занятия: время + заголовок, tap → `select-entry`.
- **Офлайн:** `readonlySlots` — без кликов, общий `opacity` на блоке календаря (как сейчас).

**Убрать:** `overflow-x-auto`, `min-w-[640px]`, `flex` из семи колонок.

**Файл:** `resources/ts/widgets/TelotronCalendar/TelotronCalendar.vue` (блок `v-else-if="mode === 'week'"`).

**Не менять в этой задаче:** месяц / день / список (agenda); календарь **клиента** (`ClientFeedCalendar`) — отдельный модуль.

---

### 3.2. Шапка страницы — компоновка

#### 3.2.1. Подзаголовок

| Действие | Текст |
|----------|--------|
| **Убрать** постоянный lead | `voiceCopy.pro.scheduleLead` — **не показывать** на странице календаря в обычном (online) состоянии |
| **Офлайн** | Достаточно `TelotronOfflineBanner` в shell (сообщение про расписание — уточнить в `telotron-theme` / `WorkspaceSchedulePage` `offlineBannerMessage` при необходимости). **Не** дублировать третьим абзацем под h1 |

Опционально в `telotron-theme.ts`: удалить или оставить ключ `scheduleLead` для других экранов — на календаре **не использовать**.

#### 3.2.2. CTA «Новое занятие» — одна строка с заголовком

**Целевой макет:**

```
┌──────────────────────────────────────────────┐
│ Календарь              [ ⊕ Новое занятие ]   │  одна строка, items-center
└──────────────────────────────────────────────┘
│ [Месяц][Неделя][День][Список]  ◀ 18–24 ▶ Сегодня │  как сейчас, в TelotronCalendar
```

| Требование | Деталь |
|------------|--------|
| **Заметность** | `variant="pill"` **или** `primary` + иконка `CalendarPlus`; min-height `var(--btn-h)` |
| **Компактность** | Не отдельный `flex` row на всю ширину под подзаголовком |
| **На узком 430px** | Допустимо: только иконка `+` с `aria-label="Новое занятие"` + текст в `sr-only`, **или** короткий текст «Добавить» — если «Новое занятие» не влезает в одну строку с h1 |
| **Офлайн** | `disabled` + визуально приглушено (как сейчас `:disabled="!online"`) |
| **Не FAB** в этой итерации | FAB перекрывает tab bar; приоритет — **header row** (проще и предсказуемо на 430px) |

**Файл:** `resources/ts/pages/workspace/WorkspaceSchedulePage.vue` (блок до `<TelotronCalendar>`).

#### 3.2.3. Дубли офлайн внутри календаря

При активном `TelotronOfflineBanner` в shell **рассмотреть** скрытие жёлтого блока внутри `TelotronCalendar` (`v-if="readonlySlots"`), чтобы не было двух жёлтых полос подряд. Если оставить один — **только shell banner**.

## 5. Критерии приёмки

### 5.1. Неделя

- [ ] Режим **«Неделя»** на ширине **430px**: **нет** горизонтального скролла календаря.
- [ ] Видны **все 7 дней** недели (Пн–Вс) без свайпа вбок.
- [ ] Раскладка: **2 колонки × 4 ряда** (7 ячеек с днями + 1 пустая или Вс `col-span-2`).
- [ ] Тап по шапке дня → режим «День» на эту дату (регрессия).
- [ ] Тап по занятию → диалог редактирования (online).
- [ ] Офлайн: слоты некликабельны, создание недоступно.

### 5.2. Шапка

- [ ] **Нет** строки «Менять расписание можно только онлайн» при online.
- [ ] **«Новое занятие»** на одной линии с заголовком «Календарь» (или icon+label при нехватке места).
- [ ] Кнопка визуально **primary**, заметна без прокрутки до календаря.
- [ ] Офлайн: одна понятная подсказка (предпочтительно shell banner), без тройного дублирования текста.

### 5.3. Регрессия

- [ ] Месяц / День / Список — без поломок.
- [ ] `npm run build` и `npm run test:ts` в контейнере Sail.

## 6. Вне scope

- Drag-and-drop перенос занятий.
- Календарь **клиента** (Client tab).
- Календарь **клиента у тренера** (`ClientFeedCalendar` — лента/месяц).
- FAB в углу экрана (можно отдельной задачей).
- Смена default mode с `week` на `month` — не требуется.

**Оценка:** **0,5–1 день** (сетка + правка шапки, без API).
