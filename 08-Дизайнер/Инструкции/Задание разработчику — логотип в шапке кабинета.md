# Задание разработчику: логотип в шапке кабинета (Pro / Client)

**От:** дизайнер (08)  
**Статус:** к передаче в разработку  
**Связь:** [Система стилей — Pro и Client (MVP) §8](Система%20стилей%20—%20Pro%20и%20Client%20(MVP).md) · [Задание — единый стиль install и auth](Задание%20разработчику%20—%20единый%20стиль%20install%20и%20auth.md) · [Стартовые выводы по бренду](Стартовые%20выводы%20по%20бренду,%20логотипу%20и%20цветам.md)

---

## 1. Проблема

После унификации **install + auth** логотип есть на публичных экранах (`TelotronPublicHeader`, `/brand/logo.png`). В **кабинете** (авторизованный shell с нижней навигацией) шапка другая: только текст «Телотрон» + подпись **Pro** / **Client**, **без** знака.

| Контур | Шапка | Логотип |
|--------|--------|---------|
| Install / login (blade) | `partials/telotron-public-header` | Да |
| SPA login / register (`meta.public`) | `TelotronPublicHeader` | Да |
| **Кабинет** (`showShell`, клиенты, календарь, «Ещё»…) | `TelotronAppShell` — текстовая ссылка | **Нет** |

Пользователь проходит **install → вход → «Клиенты» / «Главная»** и теряет узнаваемый бренд: кажется, что «вошли в другое приложение». Это расходится с каноном shell в [Система стилей §8](Система%20стилей%20—%20Pro%20и%20Client%20(MVP).md): **AppHeader: logo + zone badge + actions**.

---

## 2. Цель

1. **Один знак бренда** на всех экранах PWA после входа — как на auth и install.
2. Шапка кабинета визуально **того же семейства**, что `TelotronPublicHeader` (размер, отступы, типографика).
3. Не ломать действия справа: уведомления (заглушка), профиль, высота `--header-h`, sticky.

---

## 3. Решение (рекомендуемое)

### 3.1. Переиспользовать `TelotronPublicHeader`

Компонент уже есть: `resources/ts/widgets/TelotronPublicHeader.vue`, стили `.telotron-public-header*` в `app.css`.

**Не копировать** разметку в `TelotronAppShell` — один источник правды для logo + «Телотрон» + Pro/Client.

### 3.2. Макет sticky-шапки кабинета

Высота полосы — **`var(--header-h)`** (56px), как сейчас.

```
┌──────────────────────────────────────────────┐ 56px sticky
│ [logo 40px] Телотрон          [🔔] [👤]    │
│             Pro | Client                      │
└──────────────────────────────────────────────┘
```

| Зона | Содержимое |
|------|------------|
| **Слева** | `TelotronPublicHeader` (компактно: logo + две строки текста) |
| **Справа** | без изменений: `Button` уведомления + `RouterLink` профиль |

**Ссылка с логотипа / названия:** на «домашний» экран зоны — Pro → `workspace-clients`, Client → `workspace-home` (не пустой `/`, если редирект иной). Допустимо оставить текущий `RouterLink to="/"` **если** редирект уже ведёт на нужный экран.

### 3.3. Вариант компоновки (на усмотрение разработчика)

**Вариант A (предпочтительно):** обёртка `flex items-center justify-between` — слева header-блок, справа actions. У `TelotronPublicHeader` убрать или переопределить `mb-4` в контексте shell (сейчас класс для public-страниц с вертикальным центрированием).

**Вариант B:** вынести только «logo + тексты» в `TelotronBrandMark.vue`, использовать и в public header, и в shell — если A даёт лишние отступы.

Критерий — §5, не название варианта.

### 3.4. Что не менять

| Не трогать | Почему |
|------------|--------|
| `TelotronBottomTabBar` | Отдельная задача |
| Public-роуты (`meta.public`) | Уже с логотипом |
| Blade install/login | Уже с partial |
| Файл `/brand/logo.png` | Общий для всех зон |
| Иконки Lucide в h1 страниц | Вне scope |

### 3.5. Версия PWA

После правок layout Pro/Client поднять соответствующий **`build.pro`** / **`build.client`** в `config/version.php` (меняется shell).

---

## 4. Затрагиваемые файлы (ориентир)

| Файл | Действие |
|------|----------|
| `resources/ts/widgets/TelotronAppShell.vue` | В `v-if="showShell"` — логотип слева вместо текстового блока |
| `resources/ts/widgets/TelotronPublicHeader.vue` | При необходимости: prop `variant="inline" \| "stacked"` (без `mb-4` в shell) |
| `resources/css/app.css` | При необходимости: `.telotron-shell-header` / модификатор отступов |
| `config/version.php` | Инкремент `build.pro` и/или `build.client` |

**Опционально (не обязательно в MVP):** скриншоты в `Спецификация экранов MVP.md` — шапка с logo.

---

## 5. Критерии приёмки

- [ ] **Pro:** на экранах «Клиенты», «Календарь», «Ещё», карточка клиента, «Тренировки», «Планы питания» и т.д. в sticky-шапке виден **`/brand/logo.png`**, подпись **Телотрон** и **Pro**.
- [ ] **Client:** то же с подписью **Client** на «Главная», «Календарь», «Планы», «Тренировки» и др.
- [ ] Логотип **тот же**, что на login/register (размер ~40px, рамка/фон как `.telotron-public-header__logo`).
- [ ] Справа в шапке по-прежнему доступны **профиль** и заглушка **уведомлений**; высота шапки не «прыгает».
- [ ] Переход **login → кабинет** не ощущается сменой бренда (логотип не исчезает).
- [ ] Mobile ~430px: длинное имя не ломает layout (`min-w-0`, truncate при необходимости).
- [ ] `alt="Телотрон"` на изображении; контраст и touch targets — по [Система стилей](Система%20стилей%20—%20Pro%20и%20Client%20(MVP).md).

### 5.1. Регрессия

- [ ] Public-роуты (вход/регистрация) — шапка **не** дублируется (один header на экран).
- [ ] Offline-баннер под шапкой — без наложения на logo.
- [ ] Сборка: `npm run build` в контейнере Sail; при необходимости `npm run test:ts`.

---

## 6. Вне scope

- Новый SVG/Figma логотип (используем `logo.png`).
- Логотип в **нижней** tab bar.
- Логотип на **Filament Admin**.
- Анимация / тёмная тема логотипа.
- Полная переделка `TelotronAppShell` (только шапка).

---

## 7. Приоритет

**P1** для MVP UI: закрывает разрыв «auth с брендом → кабинет без бренда».

**Оценка:** **0,5–1 день** (переиспользование `TelotronPublicHeader` + правка отступов).
