## Контекст

Прямая регистрация на `pro.telotron.ru` / `client.telotron.ru` без invite ломает модель подключения: тренер без атрибуции, клиент без тренера.

## Решение (2026-06-16)

### Pro

- Без партнёрского токена в `localStorage` и **без** телефона → авто-редирект на лендинг `/i/{token}` ссылки **`platform_trainer_recruitment`** с title **«Прямой вход в зону pro»** (отдельно от «С сайта telotron»).
- Без токена, **с** телефоном → страница **входа**; ссылка «Регистрация» → тот же лендинг.
- С токеном → обычный поток (вход / регистрация).
- **API:** `partner_invite_token` обязателен на `POST /register/*` для zone=pro.

### Client

- Регистрация **только** по `client_onboarding` токену (ссылка тренера).
- **Вход** без токена — для уже зарегистрированных.
- **API + UI:** `/register` недоступен; `client_invite_token` обязателен на регистрации.

## Критерии готовности

- [ ] Seeder + ensure ссылки «Прямой вход в зону pro»
- [ ] `GET /api/v1/auth/pro/direct-entry-invite` → URL лендинга
- [ ] Backend: 422 без invite на register (pro/client)
- [ ] Frontend: редиректы, скрытие регистрации Client, ссылка регистрации Pro → invite
- [ ] Тесты API + Vitest; сборка PWA
