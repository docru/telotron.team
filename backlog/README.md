# Product backlog (markdown)

Единый backlog проекта по [регламенту аджайла](../01-Директор/Инструкции/Аджайл%20—%20процесс%20Telotron%20(регламент).md).

## Структура

| Путь | Назначение |
|------|------------|
| [`бэклог/`](бэклог/) | Очередь: тикет создан, ещё **не взят** в работу |
| [`в-работе/`](в-работе/) | Исполнитель **взял** тикет и ведёт его |
| [`сделано/`](сделано/) | Критерии готовности закрыты, итог в журнале тикета |
| [`отменено/`](отменено/) | Снято с плана — **только директор** |
| [`спринты/`](спринты/) | Тема спринта + ссылки на тикеты |
| [`идеи/`](идеи/) | Наброски **без спринта** и без T-XXX; созревают → [`бэклог/`](бэклог/) ([README](идеи/README.md)) |
| [`insight-brief/`](insight-brief/) | Brief раз в 2 недели (PO) — **боли с evidence**; может кормить `идеи/` или `бэклог/` |

**Один файл = один тикет** (в `бэклог/` … `сделано/`). В **`идеи/`** — файл = **идея** (без T-XXX до созревания).

## Поток: от идеи к спринту

```text
insight brief  →  идеи  →  бэклог (T-XXX)  →  спринт  →  в-работе  →  сделано
```

| Этап | Где | Кто решает «брать» |
|------|-----|-------------------|
| Набросок | [`идеи/`](идеи/) | — |
| Тикет | [`бэклог/`](бэклог/) | PO + директор |
| Спринт | [`спринты/`](спринты/) | Planning |

## Жизненный цикл тикета (папка = статус)

```text
бэклог  →  в-работе  →  сделано
                ↘
              отменено  (только директор)
```

| Действие | Кто | Что сделать |
|----------|-----|-------------|
| Новый тикет | PO / директор | Файл в **`бэклог/`**, `Статус`: `backlog` или `ready` |
| Новая идея | Любой | Файл в **`идеи/`** по [`_шаблон-идеи.md`](идеи/_шаблон-идеи.md) |
| Идея → тикет | PO / директор | Новый T-XXX в **`бэклог/`**; в идее статус `graduated` + ссылка |
| Взять в работу | **Исполнитель** | **`в-работе/`**, `Статус`: `in_progress`, запись в журнал |
| Завершить | **Исполнитель** | **`сделано/`**, `Статус`: `done`, итог в журнале |
| Отменить | **Только директор** | **`отменено/`**, `Статус`: `cancelled`, причина в журнале |

**Период 0:** gate **01.06 закрыт — Go** ([sign-off директора](../01-Директор/Sign-off%20—%20go%20период%200%20gate%2001.06%20(2026-06-11).md)). **Этап 1 · Пилот** в работе. **РКН:** подано + **внесено в реестр** ([T-016](сделано/T-016-rkn-уведомление-оператор-pd.md), [T-021](сделано/T-021-rkn-проверка-статуса-уведомления.md)).
