# T-058 · Public landing: hero-визуал (mockup / иллюстрация)

| Поле | Значение |
|------|----------|
| **Статус** | `backlog` · папка: **`бэклог/`** |
| **Приоритет** | **P2** (улучшение конверсии и восприятия; не блокирует CTA и юр. контент) |
| **Спринт** | **2–3** |
| **Исполнитель** | **дизайнер** (ассет) → **разработчик** (вёрстка) |
| **Создан** | 2026-06-16 |

> Замечание директора (2026-06-16): лендинг `telotron.ru` выглядит **техническим** — не хватает картинки/визуала в hero; только текст и иконки в карточках.

## Контекст

- Публичная страница A13 закрыта в [T-001](../сделано/T-001-a13-public-тексты-v2.md): карточки, иконки, CTA — **без** hero-иллюстрации или product mockup.
- Текущий код: `resources/views/public/welcome.blade.php`, стили `.telotron-public-landing*` в `resources/css/app.css`.
- Дизайн v2: [Задание разработчику — public A13 v2](../../08-Дизайнер/Инструкции/Задание%20разработчику%20—%20public-страница%20A13%20вёрстка%20и%20дизайн%20v2.md) — hero как текстовая карточка.
- Скрины Pro для VK — отдельно [T-056](T-056-скрины-pro-vk-пилот.md); **можно переиспользовать** кадры в mockup, если готовы.

## Проблема

| Симптом | Причина |
|---------|---------|
| Страница читается как «спецификация», не как продукт | Hero = H1 + абзац + кнопка, без визуального якоря |
| Низкая эмоциональная вовлечённость | Нет ни product mockup, ни иллюстрации, ни фото |
| Слабая дифференциация от «сырого лендинга SaaS» | Сетка иконок без контекста «как выглядит приложение» |

## Решение (рекомендация)

**Вариант A (предпочтительный):** product mockup в hero — телефон с экраном Pro (календарь или клиенты), фирменные цвета Pro, без посторонних ПДн.

**Вариант B:** мягкая иллюстрация в палитре Pro (синий + красный акцент), без фотореализма.

**Не рекомендуется для пилота:** дешёвый сток «тренер в зале» без единого стиля с продуктом.

### Макет hero

| Viewport | Компоновка |
|----------|------------|
| **Desktop (≥640px)** | Двухколоночный hero: слева H1 + lead + CTA, справа визуал |
| **Mobile** | Визуал **над** текстом (компактный crop) или сразу под H1 — без «стены текста» до скролла |

Контент §6.2 ТЗ A13 **не менять** без ревью PO; визуал — **дополнение**, не замена текстов.

## Критерии готовности

### Дизайнер

- [ ] Утверждён вариант: **A (mockup)** или **B (иллюстрация)** — зафиксировать в комментарии к тикету
- [ ] Ассеты в репозитории: `Команда/08-Дизайнер/public/` (или `_telotron.ru/public/brand/landing/` после согласования с dev)
- [ ] Форматы:
  - hero desktop: WebP + PNG fallback, ~**1200×800** (или пропорция под двухколоночный hero)
  - hero mobile: отдельный crop или тот же файл с `object-fit` — **safe zone по центру** (как VK-обложка)
- [ ] На скринах **нет** реальных ФИО/телефонов; демо-данные или обезличивание
- [ ] *(желательно)* `og:image` для шаринга — **1200×630**, тот же визуальный язык

### Разработчик

- [ ] `welcome.blade.php`: hero с визуалом (двухколоночная вёрстка на `sm+`)
- [ ] `app.css`: модификаторы `.telotron-public-landing-hero` (grid/flex, отступы, `max-width` визуала)
- [ ] `alt` у `<img>` осмысленный (не «картинка»)
- [ ] `loading="lazy"` для визуала ниже fold; hero — без lazy (LCP)
- [ ] `PublicWelcomePageTest` green; при добавлении `og:image` — assert или отдельный smoke
- [ ] Проверка **360px** и **1280px**: без горизонтального скролла, CTA заметен

### Приёмка

- [ ] Локально `telotron.test`: hero не «технический», визуал читается за 2–3 сек
- [ ] Smoke prod `telotron.ru` после деплоя
