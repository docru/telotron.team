## Контекст

**Задача продукта:** партнёрка — **основной канал** привлечения тренеров ([матрица тарифов](../../../_telotron.ru/docs/Бизнес-требования/00-канон-mvp/матрица-тарифов-платформа.md), [модуль-партнерка](../../../_telotron.ru/docs/Бизнес-требования/02-модули/partner/модуль-партнерка.md)). Сейчас партнёрская ссылка (`specialist_referral`, `/i/{token}`) **есть**, но **спрятана** за вторым блоком в диалоге приглашений — тренер чаще делится **клиентской** ссылкой и не думает о реферальной.

**Идея директора (12.06.2026):**

1. **Рядом с приглашением клиента** — показать партнёрскую ссылку с **мотивационной меткой** (не слово «скидка» — см. § «Формулировки по статусу»).
2. **На странице оплаты** (Commerce) — блок с призывом поделиться ссылкой; текст **зависит от статуса коммерции** (платный тариф vs Лайт/триал).

**Связь с пилотом:** до **01.08** оплат нет — UI может быть **заложен**; финальные тексты и показ CTA про экономию — когда Commerce и партнёрские начисления **работают**, либо с нейтральной формулировкой «с августа» (см. вопросы).

**Текущий код (точки входа):**

- `ProInviteDialog.vue` — диалог «Пригласить клиента» / «Поделиться с другим тренером» (два секции через `Separator`).
- `ProInvitesPanel.vue` — страница «Ещё» с двумя карточками.
- `WorkspaceClientsPage.vue` — sticky CTA «Пригласить клиента» → `workspace-more`.
- Commerce UI checkout — **ещё не реализован** ([commerce-модуль-тз-mvp §9](../../../_telotron.ru/docs/Техдок/03-модули/commerce-модуль-тз-mvp.md)).

---

## Цель (после обсуждения)

**Грамотно подталкивать** тренера к шерингу **партнёрской** ссылки, не путая с клиентской, **без** нарушения юр. рамок пилота и **без** ложных обещаний дохода. Мотивация — **снижение расходов на абонплату** (не ярлык «скидка»).

**Не цель:** агрессивный upsell, pop-up при каждом входе, спам в мессенджеры.

---

## Формулировки по статусу коммерции (решение директора, уточнить при реализации)

**Принцип:** не использовать слово **«скидка»** в UI nudge. Текст badge / подзаголовка **зависит от** `commerce_status` и **тарифа** (`effective_tariff`).

| Сегмент | Когда | Черновик метки / заголовка | Смысл |
|---------|--------|----------------------------|--------|
| **Платные тарифы** | `active`, тариф **Профи** или **Максимальный**, тренер **платит** | **«Снизить месячные платежи»** | Партнёрское вознаграждение зачитывается в счёт абонплаты ([партнёрка §4](../../../_telotron.ru/docs/Бизнес-требования/02-модули/partner/модуль-партнерка.md), до **S**) |
| **Триал** | `trial` | **«Как снизить платежи»** *(или короче)* | Объяснить механику **до** первой оплаты; без обещания «бесплатно навсегда» |
| **Лайт** | `light` | **«Как снизить платежи»** **или** **«Как подключить „Профи“ и не платить»** *(рабочий вариант)* | Акцент на выходе с Лайта / оплате через партнёрскую программу; **не** «скидка» |

**При реализации уточнить:**

- Точные строки под **Voice Pro** и названия тарифов в UI («Профи» vs «Профессионал» — [матрица тарифов](../../../_telotron.ru/docs/Бизнес-требования/00-канон-mvp/матрица-тарифов-платформа.md)).
- Tooltip / раскрытие одной строкой («когда коллега пополняет счёт…») — PO + юрист.
- **Заморозка** (`frozen`) — в группу платных или отдельная нейтральная формулировка.
- До **01.08** — показывать нейтральный вариант («Партнёрская программа» / «Как это работает») или скрывать экономический hook.

**Визуал badge:** по-прежнему **мелкий акцент** (например `text-destructive` / primary link) — **без** слова «скидка».

## Предложение UX (черновик для макета)

### A · Диалог / экран приглашения (Pro)

```
┌─ Пригласить клиента ─────────────────────┐
│  [ ссылка client_onboarding    ] [📋][↗]│
│                                          │
│  Пригласить коллегу-тренера              │
│  [Снизить месячные платежи]  ← badge     │  платный тариф (черновик)
│  [ ссылка specialist_referral  ] [📋][↗]│
└──────────────────────────────────────────┘
```

- Primary — **клиент**; партнёрский блок **сразу под** клиентским (не за fold / не на второй вкладке).
- **Badge** — мелкий акцент (`text-xs`); текст **по сегменту** (§ «Формулировки»), не «скидка».
- Tooltip / подпись — одна строка про зачёт вознаграждения в абонплату (черновик, **уточнить при реализации** + юрист).
- Копирование / Share — как сейчас.

**Файлы:** `ProInviteDialog.vue`, `ProInvitesPanel.vue` (+ при необходимости sticky CTA на `WorkspaceClientsPage`).

### B · Страница оплаты (Commerce, §9 commerce TЗ)

После выбора пакета / перед redirect в ЮKassa — **вторичный** блок (не блокирует оплату). **Заголовок по сегменту:**

| Сегмент | Черновик заголовка блока |
|---------|--------------------------|
| Платный (`active`, pro/max) | **Снизить месячные платежи** |
| Триал | **Как снизить платежи** |
| Лайт | **Как снизить платежи** *или* **Как подключить «Профи» и не платить** |

> Поделитесь партнёрской ссылкой с коллегами-тренерами. *(пояснение — юрист, при реализации)*  
> [ ссылка ] [Копировать] [Поделиться]

- Показывать, если есть активный `specialist_referral` token.
- До **01.08** — нейтральный вариант или скрытие экономического hook (**Q-P02**).

## Юридические ограничения (обязательно)

- **Пилот до 01.08:** в публичных текстах **не обещать** выплаты и зачёт в абонплату как уже действующие ([План продвижения §1.4](../../02-Маркетолог/План%20продвижения%20—%20этап%201%20Пилот.md)).
- **In-app после 01.08:** формулировки согласовать с юристом; опираться на [модуль-партнерка §4](../../../_telotron.ru/docs/Бизнес-требования/02-модули/partner/модуль-партнерка.md) (зачёт **до S**, остаток на внутренний счёт).
- **Не использовать в UI nudge:** слово **«скидка»**, «гарантированный доход», «пассивный заработок», «бесплатно навсегда».
- **Особый риск:** «подключить Профи и **не платить**» — только после юр. sign-off (**Q-P03**).
