## 1. Видео упражнений для конструктора упражнений

**Суть:** к упражнению в **конструкторе** (библиотека / план) — **видео**: техника выполнения, демонстрация. Тренер выбирает из каталога или прикрепляет своё; клиент видит в плане.

**Зачем:** из интервью MVP — видео техники не «nice-to-have», а базовая ценность ([MVP-2](../../../Команда/03-Разработчик/Исследования/Интервью/MVP/MVP-2.md), [MVP-3](../../../Команда/03-Разработчик/Исследования/Интервью/MVP/MVP-3.md)).

**Открыто:** хранение (S3/CDN), лимиты размера, модерация, авторские права, тариф (все / Профи / Макс).

---

## 2. Телемост для проведения видеозанятий

**Суть:** встроенная или интегрированная **видеосвязь для занятий** — вероятно **Яндекс Телемост** (или аналог): ссылка из записи в расписании, «начать занятие» для тренера, вход клиента.

**Зачем:** онлайн-тренировки без ухода в Zoom/TG; единый контур «запись → занятие».

**Открыто:** нативный UI vs deep link; запись эфира; ПДн и юр.; оплата онлайн-занятия (A2); **M8** vs отдельный модуль **M15** интеграции.

---

## 3. Встроенный мессенджер для общения с клиентом

**Суть:** **чат тренер ↔ клиент** внутри Pro/Client — не MAX/TG как основной канал переписки по тренировкам, а продуктовый мессенджер (история, вложения, привязка к клиенту/записи).

**Зачем:** позиционирование «без хаоса в чатах» ([VK-план](../../../Команда/02-Маркетолог/Исследования/Маркетинг-план%20на%20пилот/04-продвижение-vk.md)); [MVP-1](../../../Команда/03-Разработчик/Исследования/Интервью/MVP/MVP-1.md) — минимальный чат в MVP-1 Plus.

**Связь:** **M3** в roadmap; напоминания **M4** (MAX/TG) — канал уведомлений, не замена чата. Мед. ПДн в чате — [юр. контур](../../../Команда/07-Юрист/Схема%20юрдокументов%20и%20этапы%20акцепта%20(Pro%20·%20Client).md).

**Открыто:** real-time (WebSocket) vs polling; файлы; read-only при даунгрейде тарифа ([матрица тарифов](../../../_telotron.ru/docs/Бизнес-требования/00-канон-mvp/матрица-тарифов-платформа.md)).

---

## 4. Статьи · обучающий контент для клиента

**Суть:** **статьи / обучающие материалы**, которые тренер **отправляет клиенту** (или добавляет в план): питание, восстановление, техника — библиотека платформы + свои тексты тренера.

**Зачем:** дополнение к плану тренировок; «кинул ссылку» вместо PDF в мессенджере; upsell контентом на Профи/Макс.

**Открыто:** редактор (Markdown/WYSIWYG); шаблоны от Telotron; Client UI «материалы от тренера»; SEO публичных статей vs private share.

---

## Общая гипотеза

Если тренер **не уходит в TG/Zoom** для видео, чата и материалов — выше **activation и retention**; проверка — интервью пилота, доля клиентов с ≥1 сообщением/материалом в продукте.
