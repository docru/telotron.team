## Суть

**Оплата налогов для самозанятых** тренеров через **интеграцию с API ФНС** (приложение «Мой налог» / партнёрский контур) — когда в продукте накопится **достаточный объём** операций и запросов.

Не замена бухгалтерии целиком, а **удобный слой**: напоминания, расчёт к уплате, формирование чека/платежа, статус «оплачено» — в связке с [Commerce](../../../_telotron.ru/docs/Техдок/03-модули/commerce-схема-данных-mvp.md) и доходами тренера в Telotron.

## Зачем / для кого

- **Тренер-самозанятый** — меньше ручной рутины вне продукта.
- **Telotron** — дифференциатор vs «таблица + мессенджер»; удержание на **Макс** или отдельный add-on.
- **Юрист / бух** — только после объёма и формализации партнёрства с ФНС/агрегатором.

## Триггер «достаточный объём»

Идея **не в ближайший спринт**. Запускать интеграцию, когда выполнено хотя бы одно (TBD с директором):

- [ ] N активных самозанятых тренеров на платформе (например **≥50**?)
- [ ] M ₽/мес проходит через Commerce у самозанятых
- [ ] ≥K обращений в поддержку «как платить налог» / insight brief

## Гипотеза

Если налоги самозанятого **внутри** Telotron (после порога объёма), снижается отток solo-тренеров на Профи/Макс; метрика — retention + NPS среди самозанятых.
