## Суть

**Индикатор (сигнал / риск-скор)**, что клиент **скоро «сольётся»** — перестанет ходить, отвалится из воронки, не продлит. Тренер видит **кто под угрозой** и **почему**, до факта ухода.

**Примеры сигналов (черновик):**

- давно не было записей / занятий vs обычный ритм;
- пропуски подряд, отмены без переноса;
- нет входа в Client / не открывает план;
- нет ответов в переписке (когда будет [мессенджер](контент-коммуникации-обучение.md));
- падение активности в дневнике / трекере (когда будет M7);
- «молчит» N дней после триала / после последней оплаты (A2 + Commerce).

**UI:** бейдж «риск», список «требуют внимания», подсказка «что сделать» (написать, предложить слот, созвон).

## Зачем / для кого

- **Тренер** — удержание дешевле привлечения; проактивный контакт вместо «пропал».
- **Telotron** — дифференциатор CRM/Макс; метрика retention на пилоте.
- **PO** — definition «слива» для activation/KPI ([T-004](../бэклог/T-004-оценка-привлечения-воронка.md)).

## Гипотеза

Если тренер **заранее** видит риск и делает 1 касание, часть клиентов остаётся; проверка — A/B на пилоте: группа с индикатором vs контроль (retention 30/60 дней).
