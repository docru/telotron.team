## Контекст

Для **личных касаний** и первых регистраций в пилоте тренеру нужен **один понятный документ**: от перехода по партнёрской ссылке до первого клиента и занятия в календаре.

**Не in-app онбординг** — это маркетинговый артефакт для отправки (PDF). Канон текстов UI позже — [T-060](../в-работе/T-060-pro-onboarding-k-p.md); ориентир по смыслу шагов — эта инструкция.

**KPI пилота (activation):** реальный клиент + занятие **или** план — те же шаги, что в чеклисте инструкции.

## Критерии готовности

- [x] Текст инструкции: шаги **1–8**, блок «Если ведёте группы», чеклист «Минимум за первые 2 дня», «Чего пока нет», обратная связь, контакты
- [x] **10 скринов** prod в `Команда/02-Маркетолог/Инструкции/скрины/` (`01` … `12`), подписи в README
- [x] Архив скринов с осмысленными именами: `Команда/02-Маркетолог/Скрины/`
- [x] Сборка **воспроизводима**: `build-onboarding-docx.py` → `.docx` + `.pdf`
- [x] PDF **15 стр.**: титульная (логотип Pro, цвета бренда, пилот до **31.07.2026**, `pro.telotron.ru`) + контент без дублирующего H1
- [x] Скрины в PDF **читаемы** (масштаб ~25% от 720×1600); **нет** пустых страниц
- [x] Таблицы pandoc в PDF заменены на **списки** (вступление, чеклист, «Чего пока нет», контакты)
- [x] Блок «Для команды (не отправлять тренеру)» **не** попадает в PDF
- [x] **Канон отправки:** PDF, не `.docx` (в Word/LibreOffice embed картинок ненадёжен)
