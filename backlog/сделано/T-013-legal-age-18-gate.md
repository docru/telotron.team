## Задачи

| # | Задача |
|---|--------|
| 1 | **RegisterPage** (Pro + Client): если `birth_date` **не** собирается на регистрации — обязательная галка «Подтверждаю, что мне исполнилось **18 лет**» (отдельно от юр. галки на 3 документа) |
| 2 | Если на регистрации появится поле даты рождения — при возрасте **&lt; 18** блок с текстом «Сервис предназначен для пользователей 18+» |
| 3 | **API + UI профиля Client:** `PATCH /api/v1/me/client-profile` — отклонять `birth_date`, дающую возраст &lt; 18 (**422**) |
| 4 | **Register API** (`SpaPasskeyRegisterController` или шаг до verify): серверная проверка галки 18+ / возраста (не только фронт) |
| 5 | Feature-тесты: галка обязательна; &lt; 18 в профиле; happy path 18+ |

## Критерии готовности

- [x] Pro/Client: нельзя завершить регистрацию без подтверждения 18+ (или без валидной даты ≥ 18)
- [x] Client profile: сохранение `birth_date` &lt; 18 → 422, понятное сообщение
- [x] Тесты в `tests/Feature/Api/` + Vitest (`meetsServiceMinimumAge`)
